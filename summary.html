total-recall

An AI sales engineer that learns and improves across sessions — using Anthropic's Managed Agents memory store to retain institutional knowledge between calls.

Same question. Two sessions. The agent answers better the second time — because it remembered.

Pipeline Core Build

Step 1

Provision Agent

Create the agent, environment, and memory store. IDs are saved to .agent_id, .environment_id, and .memory_store_id for reuse.


python create_agent.py
Step 2 — Session 1 (Baseline)

First pitch strategy session

Agent reads Round 1 docs (Northwind's stack, prior objections, pitch deck), answers the test question, and saves key facts to memory.


python run_session_1.py → outputs/session1.txt
Persistent Memory Store — /mnt/memory/

Mounted read/write in every session. Agent organizes its own files. New sessions read this store before doing anything else.

prospect_environment.md
Northwind's stack, constraints, residency
objections.md
All objections + latest answers
competitive_intel.md
Competitor updates, dated entries
pitch_strategy.md
Current approach, revision history
contacts_and_roles.md
Key people, titles, concerns
Step 3 — Session 2 (Learning)

Session with new contradictions

Agent reads memory first, then round 2 docs with a new CISO blocker and competitive updates. It reconciles conflicts — updates, doesn't append — then answers sharper.


python run_session_2.py → outputs/session2.txt
Step 4 — Compare

Before vs. after — the proof

Side-by-side diff reveals the agent anticipates the new objection, cites the new capability, and resequences the pitch — because it remembered.


diff outputs/session1.txt outputs/session2.txt

Stretch Goals S1 – S9

Designed

S1 — Memory Policy

Explicit ALWAYS/NEVER lists in the system prompt for what to memorize.

Implemented

S2 — Memory Curator

A Haiku 4.5 sub-agent that periodically cleans the memory store: merges duplicates, flags conflicts, prunes stale entries.

Implemented

S3 — Adversarial Round

Undated contradictions arrive. Agent must flag the uncertainty rather than silently overwriting.

Implemented

S4 — Introspection

No new docs. Agent is asked: "What have you learned?" — pure memory recall and synthesis.

Designed

S5 — Per-Tenant Memory

Isolated memory stores per customer_id. Northwind and Globex never share knowledge.

Designed

S6 — Scheduled Ingestion

Daily inbox scan automatically adds to memory — no human in the loop.

Designed

S7 — Long-Context Compaction

20+ turn session; agent does in-session memory management and writes a summary to the store at the end.

Designed

S8 — Memory Diff View

Snapshot memory before and after a session; surface a visual diff of what changed.

Designed

S9 — Topic Sub-Agents

Expert routing: a router agent dispatches to specialized agents per memory category (competitive, technical, legal).

Tech Stack

🤖

Managed Agents

Cloud-hosted agent runtime. Handles environment lifecycle, session isolation, and tool execution.

💾

memory_stores API

Path-organized filesystem mounted at /mnt/memory/ — persists across sessions, agent writes its own schema.

⚡

Claude Sonnet 4.6

Main agent model. Claude Haiku 4.5 used for the memory curator sub-agent (S2) for cost efficiency.

🐍

Anthropic Python SDK

client.beta.sessions — streaming events API for interactive multi-turn sessions.
