Mocked seam. In the real build, a Managed Agent with the Memory tool holds one memory file per person (scoped by user ID), and a memory-curator sub-agent reads new Slack activity (#furballs, #sf_office, #shoutouts) plus the Notion projects DB and decides what is quiz-worthy vs. noise. This page has no server and no agent: the memory below is hardcoded, and “Simulate new Slack activity” replays one scripted update client-side to show the demo diff.
